Как работает Jorenza

Полное руководство по использованию платформы для деплоя GitHub проектов.

🚀 Быстрый старт

Jorenza превращает любой публичный GitHub репозиторий в живой сайт за несколько секунд. Никаких конфигов, CLI или токенов — просто ссылка.

Шаг 1: Открой приложение

Перейди на страницу приложения или воспользуйся полем на главной странице.

Шаг 2: Вставь ссылку на репозиторий

Скопируй URL своего публичного GitHub репозитория. Он выглядит так: https://github.com/username/repo-name

Шаг 3: Нажми Deploy

Jorenza автоматически:

  • Получит информацию о репозитории через GitHub API
  • Построит дерево всех файлов проекта
  • Сгенерирует raw-ссылки на каждый файл
  • Выдаст уникальный URL вида jorenza.app/p/xK9mN2pQaB
  • Создаст короткую ссылку jrn.to/xK9mN2 для шеринга

🔗 Система ссылок

Jorenza использует GitHub Raw Content API для прямого доступа к файлам репозитория.

Формат Raw URL

Каждый файл в репозитории получает прямую ссылку вида:

https://raw.githubusercontent.com/{owner}/{repo}/{branch}/{path}

Например, файл src/index.js в репозитории user/my-app ветки main:

https://raw.githubusercontent.com/user/my-app/main/src/index.js

Уникальный URL деплоя

Каждый деплой получает уникальный идентификатор из 10 символов (буквы и цифры), исключающих визуально похожие символы (0/O, 1/l/I):

jorenza.app/p/xK9mN2pQaB

Короткая ссылка

Для удобного шеринга в соцсетях генерируется короткий код из 6 символов:

jrn.to/xK9mN2

⚙️ Поддерживаемые типы проектов

Static HTML/CSS/JS

Простейший случай — статические сайты. Jorenza находит index.html как точку входа и раздаёт все связанные файлы через raw ссылки.

React (Vite / Create React App)

Определяется по наличию package.json, файлов .jsx или .tsx. Jorenza показывает структуру проекта и предоставляет ссылки на все исходники.

Vue.js

Определяется по файлам .vue или vue.config.js.

Next.js

Определяется по next.config.js или next.config.mjs.

Другие типы

Python, Go, TypeScript — Jorenza определит язык и тип проекта, предоставит полную карту файлов с raw-ссылками.

🆔 ID проекта

Каждый деплой на Jorenza имеет уникальный идентификатор. Текущий ID проекта Jorenza: JRZ-001

ID деплоев хранятся локально в браузере пользователя и используются для восстановления истории деплоев.

⚠️ Ограничения

  • Только публичные репозитории GitHub
  • Максимум 150 файлов отображается в дереве (у больших репо ограничено для производительности)
  • Серверные проекты (Node.js бэкенды, Python Flask/Django и т.д.) — только просмотр файлов, не запуск сервера
  • GitHub API Rate Limit: 60 запросов в час без токена
